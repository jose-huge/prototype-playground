# Built component file placement

When building a new component for the playground, files MUST go here:

| File | Location |
|---|---|
| Component | `/components/{ComponentName}.tsx` |
| CSS module | `/components/{ComponentName}.module.css` |

**Never place built components in `/app/components/`** — that directory contains playground infrastructure (Playground, BuildPanel, WorkBench, etc.) and is never scanned for built components.

**Never manually create a preview page.** The route `/api/components` scans `/components/` and auto-generates `/app/preview/component/{Name}/page.tsx` on every call. The playground calls this automatically.

**Never manually edit `/components/index.ts`.** That barrel file is also auto-generated by `/api/components`.

# Built component authoring rules

**Exports (required):**
- Use a named export: `export function ComponentName(...)`
- Do NOT use `export default`
- Export the color scheme: `export const defaultScheme = "{scheme-name}" as const;`
  Set the scheme to match the Figma frame (e.g. `"white"`, `"harvest-light"`, `"dark"`)

**Color scheme:**
- Do NOT put `data-scheme` on the component's root element
- The playground wraps the component in a `SchemeProvider` that applies `data-scheme` via a `display:contents` ancestor — no scheme hook or override logic is needed in the component
- Do NOT import or use `useSchemeOverride`

**"use client" directive:**
- Add `"use client"` ONLY if the component uses React hooks (`useState`, `useEffect`, `useRef`, etc.) or browser APIs
- Pure presentational components that receive all data via props do NOT need `"use client"`
- Do NOT add `"use client"` just for scheme handling

**Component imports:**
- Use UI primitives from `@/components/ui/` (buttons, inputs, etc.) where they apply
- Do NOT import other built components from `/components/`

**Tokens and responsive behavior:**
- Design system tokens handle fluid sizing via CSS container query units — apply them automatically
- Only add `@container` or `@media` rules if the layout structure itself changes at a breakpoint
